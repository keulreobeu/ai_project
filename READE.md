# LocalHub 개발 의뢰서 요약

**간단요약:** LocalHub는 제공된 공공데이터 기반 익명 커뮤니티(익명 게시판 + 챗봇)를 Vue.js 3 + FastAPI + SQLite 스택으로 개발하는 과제입니다. 필수 기술 스택과 배포(Netlify/Render)가 지정되어 있으며, 민감정보는 반드시 `.env`로 관리해야 하고, 추가 데이터 사용 시 라이선스(공공누리 유형)를 사전 확인해야 합니다. 개발 환경은 VSCode Copilot + OpenAI API만 허용되며, 납기는 2026-07-16 15:00입니다.

## 정리 기준
- 구분: 필수 / 제약 / 선택
- 항목: 요구사항 명칭
- 상세내용: 구현 또는 준수해야 할 내용
- 근거 조항: RFP 문서의 해당 위치
- 검토 의견: PM 관점 주요 확인 사항

## 추가 검토 포인트
- 선택 기능 구현 시에도 추가 데이터 사용은 반드시 라이선스 검토가 필요함.
- `.env` 파일은 반드시 `.gitignore`에 등록하고, 제출 시 포함되지 않도록 확인해야 함.
- 개발 도구 정책을 위반할 경우 계약 이슈가 발생하므로, 외부 AI 코딩 도구 사용 금지는 절대 준수해야 함.

- 외부 AI 도구 사용 위반: Claude Code/Codex/Cursor/Antigravity 등 금지된 도구 사용 시 계약 조건 위반.
- 추가 데이터 라이선스 미확인: 선택 기능이나 외부 API 추가 시 라이선스/공공누리 유형을 확인하지 않으면 법적/계약적 문제가 발생.
- 배포 실패 또는 URL 미확인: Netlify/Render 배포 후 URL이 정상 동작하지 않으면 제출물 불완전 처리 가능.
- 일정 지연: 2026-07-16 15:00 제출 기한이 엄격하므로 기능 범위와 일정 계획을 빠르게 확정할 필요.
- 평문 비밀번호 저장: 요구사항대로 구현하더라도 실제 서비스 보안 취약성이 있으므로 내부 검토 시 위협을 명시해야 함.

## 완료 우선순위
1. 필수 CRUD 기능 및 Vue.js 화면 구현
2. FastAPI + SQLite API 서버 구축 및 DB 스키마 설계
3. 챗봇 API `/api/chat` 및 제공 JSON 기반 질의응답 처리 구현
4. 민감 정보 `.env` 관리, `.gitignore` 설정, 코드 내 비밀정보 노출 방지 검토
5. Netlify/Render 배포 환경 구성 및 배포 URL 확인
6. 제출 산출물 준비: 프론트엔드/백엔드 repo URL, DB 파일, 기능 명세서, WBS, 발표 자료
7. 선택 기능 중 하나 이상 추가 구현 (지도, 데이터 시각화, 캘린더 등)
8. 데이터 출처·라이선스 목록 정리 및 기능 명세서에 별도 항목 포함
9. 테스트 및 QA: 배포 URL, 챗봇 응답, CRUD 동작, 모바일 UI, 플로팅 챗봇 검증
